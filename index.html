CodeMirror 2

This is a project aiming to create a CodeMirror-style component without using an IFRAME. CodeMirror 1, with its editable frame, still works pretty well but...

  • the lack of control over the actual frame content makes some things really hard to do and/or slow
  • the frame makes the API unnecessarily awkwardâ€”it has to load asynchronously, and, for example, can't be moved around the DOM
  • editable elements are a shady area of browser functionality that is full of bugs
  • (the same goes for controlling selection in arbitrary DOM structures)
  • some of the design decisions I made in the original implementation make things slower than they have to be
  • the complexity of the code, both from work-around cruft and from the inherent trickiness of having the DOM be your data representation, make it hard to modify at this point

So this is a from-scratch implementation, using non-editable DOM elements for presentation, and a hidden textarea for input and local cursor motion. It aims to be:

  • really small
  • simple to use (almost no custom keybindings)
  • so generic that it can be embedded everywhere
  • fast enough to work with huge files (on modern browsers)

One thing it won't do is support line-wrapping. I've spent a few days investigating this, and had to conclude that the slowdown and increased flakiness of supporting line-wrapping isn't going to be worth it.
